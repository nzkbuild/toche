# Toche 1.1.0

Toche 1.1.0 transforms Toche from a Claude Code-specific local gateway into a
safe multi-client AI workload runtime. One local Toche instance can now serve
Claude Code and Codex CLI simultaneously, with trust-domain isolation preventing
cross-client cache or flight sharing.

## Install

Publish this GitHub Release before publishing the npm package. Once both are
public, install Toche with:

```shell
npm install -g @nzkbuild/toche
toche setup
toche
```

Then launch your clients:

```shell
claude
codex
```

The npm package name remains `@nzkbuild/toche`. The installed command remains
`toche`.

## What changed

### Multi-client runtime

One `toche` process now accepts connections from multiple simultaneous AI
clients. Claude Code and Codex CLI can run concurrently through the same local
instance.

- **New:** `toche status` reports live runtime state including active integrations, flights, and waiters.
- **New:** `toche stats` filters by `--protocol`, `--integration`, and `--trust-domain`.
- **New:** `toche run <client>` launches a client through Toche in one step (managed mode).
- **New:** `toche connect` and `toche disconnect` accept `claude` or `codex` arguments.

### Configuration v2

The overloaded `Profile` concept is replaced with separate `RuntimeConfig`,
`Integration`, `Upstream`, `Policy`, and `StorageConfig` types. Deterministic
secret references keep credentials out of logs and receipts.

- `profiles.toml` → `config.toml`: migration is automatic, transactional, idempotent, and backed up.
- `toche setup` is now rerunnable, previews changes, and modifies only Toche-owned fragments.
- Setup detects Claude Code and Codex CLI installations and configures them independently.

### Trust-domain isolation

Requests with different credentials, integrations, or privacy policies no longer
share cache entries or in-flight request coalescing. Leader panics clean up
without corrupting waiters, and waiter cancellation leaves the leader intact.

### Protocol-driver architecture

Anthropic Messages retains the complete existing optimization pipeline. OpenAI
Responses (`/v1/responses`) has correct pass-through forwarding, streaming, trust
isolation, and unknown-field preservation. No cross-protocol translation occurs.

### Upgrade from 1.0.x

Existing `1.0.10` configurations migrate automatically on first run. The
persistent two-terminal workflow (`toche` + `claude`) remains first-class.
Managed mode is optional.

## Manual npm publication

After this GitHub Release and all native assets are public, check out the exact
`v1.1.0` tag and run:

```shell
npm whoami
npm run test:npm
npm pack --dry-run
npm publish --access public
```

`npm whoami` must print `nzkbuild`.
