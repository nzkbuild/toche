# Toche 1.0.9

Toche 1.0.9 corrects the npm package identity after npm rejected the unscoped
`toche` name as too similar to existing packages. The native command remains
`toche`.

## Install

Publish the GitHub Release before publishing the npm package. Once both are
public, install Toche with:

```shell
npm install -g @nzkbuild/toche
toche setup
toche
```

The npm installer downloads the matching native 1.0.9 archive and verifies its
SHA-256 checksum before exposing the `toche` command.

Supported npm targets are Windows x64, Linux x64, Intel macOS, and Apple silicon
macOS. Node.js 18 or newer is required for installation. Toche itself runs as a
native Rust binary.

## What changed

- The public npm package is now `@nzkbuild/toche`.
- The installed command remains `toche`.
- Installation, removal, publishing, and recovery instructions use the scoped
  package consistently.
- Release metadata and package tests now target 1.0.9.

No gateway behavior, cache policy, request transformation, or local data format
changed in this patch.

## Manual npm publication

After this GitHub Release and all native assets are public, check out the exact
`v1.0.9` tag and run:

```shell
npm whoami
npm run test:npm
npm pack --dry-run
npm publish --access public
```

`npm whoami` must print `nzkbuild`.
