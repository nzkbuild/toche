# Toche 1.0.6

This is the first public GitHub Release for Toche, a local efficiency gateway for
Claude Code. It packages the completed 1.0 line together with the fixes and
verification work through 1.0.6.

## What Toche does

- Coalesces identical in-flight requests so they can share one upstream response
- Coordinates Anthropic prompt-cache breakpoints in observe or automatic mode
- Reduces known noise from tool output while preserving the original content for recovery
- Stores eligible text-only responses in a workspace-aware persistent safe cache
- Records usage, cache behavior, reduction savings, latency, and estimated cost locally
- Provides efficiency profiles, session checkpoints, bypass controls, and an optional Graphify adapter

## Install

Download the archive for your platform:

- `toche-1.0.6-x86_64-pc-windows-msvc.zip`
- `toche-1.0.6-x86_64-unknown-linux-gnu.tar.gz`
- `toche-1.0.6-x86_64-apple-darwin.tar.gz`
- `toche-1.0.6-aarch64-apple-darwin.tar.gz`

Each archive has a matching `.sha256` file. Verify the checksum before running
the binary.

You can also build from source with Rust 1.85 or newer:

```bash
cargo build --release --locked
```

Then run `toche setup`, start the gateway, run `toche connect` in another
terminal, and confirm the routing state with `toche doctor`.

## Safety model

Toche runs as a local gateway and keeps its configuration, ledger, caches,
checkpoints, and content-addressed storage on the machine where it runs. Requests
that require upstream work still go to the Anthropic API endpoint configured in
the active profile.

Persistent replay rejects responses containing `tool_use` blocks. Optimization
stages have explicit bypass headers, reduced output is recoverable with
`toche expand`, and `toche disconnect` restores direct Claude Code routing.

## Verification history

The 1.0 line received several rounds of internal codebase auditing and dogfooding.
The 1.0.6 source commit reports 354 passing tests, zero compiler warnings, and
four working Criterion benchmarks. The release workflow reruns formatting,
Clippy, and the full test suite against the tagged source before creating assets.

This is not a claim of an independent third-party security audit.
