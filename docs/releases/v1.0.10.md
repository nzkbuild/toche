# Toche 1.0.10

Toche 1.0.10 fixes npm installation on Windows. Version 1.0.9 downloaded and
verified the correct release archive, but passed its extraction paths to
Windows PowerShell incorrectly. That left `Expand-Archive` with an empty input
path and caused npm to remove the incomplete installation.

## Install

Publish this GitHub Release before publishing the npm package. Once both are
public, install Toche with:

```shell
npm install -g @nzkbuild/toche
toche setup
toche
```

The npm package name remains `@nzkbuild/toche`. The installed command remains
`toche`.

## What changed

- Windows archive and destination paths are passed to PowerShell through
  environment variables instead of unreliable positional arguments.
- Windows CI now creates and extracts a real ZIP archive through paths that
  contain spaces.
- The release workflow must pass the Windows installer regression test before
  it can create a tag or draft release.

No gateway behavior, cache policy, request transformation, native binary, or
local data format changed in this patch.

## Manual npm publication

After this GitHub Release and all native assets are public, check out the exact
`v1.0.10` tag and run:

```shell
npm whoami
npm run test:npm
npm pack --dry-run
npm publish --access public
```

`npm whoami` must print `nzkbuild`.
