# Toche 1.0.8

This is the first public GitHub Release for Toche, a local efficiency gateway for
Claude Code. It packages the completed 1.0 line, the npm installer, and the
release-readiness fixes through 1.0.8.

## What Toche does

- Coalesces identical in-flight requests so they can share one upstream response
- Coordinates Anthropic prompt-cache breakpoints in observe or automatic mode
- Reduces known noise from tool output while preserving the original content for recovery
- Stores eligible text-only responses in a workspace-aware persistent safe cache
- Records usage, cache behavior, reduction savings, latency, and estimated cost locally
- Provides efficiency profiles, session checkpoints, bypass controls, and an optional Graphify adapter

## Install

The shortest installation path is npm:

```shell
npm install -g toche
toche setup
toche
```

Open a second terminal and run `toche connect`, followed by `toche doctor`. The
npm installer downloads the matching native release archive and verifies its
SHA-256 checksum before exposing the `toche` command.

The supported npm targets are Windows x64, Linux x64, Intel macOS, and Apple
Silicon macOS. Node.js 18 or newer is required for installation, but Toche itself
runs as a native Rust binary.

For manual installation, download the archive for your platform:

- `toche-1.0.8-x86_64-pc-windows-msvc.zip`
- `toche-1.0.8-x86_64-unknown-linux-gnu.tar.gz`
- `toche-1.0.8-x86_64-apple-darwin.tar.gz`
- `toche-1.0.8-aarch64-apple-darwin.tar.gz`

Each archive has a matching `.sha256` file. Verify the checksum before running
the binary.

The draft also contains `toche-1.0.8.tgz`, the package that will be published to
npm after the native release assets are public.

You can also build from source with Rust 1.86 or newer:

```shell
cargo build --release --locked
```

- Release packaging uploads only completed archives and checksums, never build directories.
- Existing tags are never moved when a newer release candidate is prepared.

## Safety model

Toche runs as a local gateway and keeps its configuration, ledger, caches,
checkpoints, and content-addressed storage on the machine where it runs. Requests
that require upstream work still go to the Anthropic API endpoint configured in
the active profile.

Persistent replay rejects responses containing `tool_use` blocks. Optimization
stages have explicit bypass headers, reduced output is recoverable with
`toche expand`, and `toche disconnect` restores direct Claude Code routing.

## Verification

The release workflow reruns npm package tests, formatting, Clippy, and the full
Rust test suite before it creates the tag or any release assets. The GitHub Release
remains a draft until all platform archives, checksums, and the npm tarball receive
a human review.

The full 354-test Rust suite passed on minimum Rust 1.86.0, Linux stable, and
Windows before this release candidate was prepared. The npm installer tests and
package-content inspection also passed in CI.

The 1.0 line received several rounds of internal codebase auditing and dogfooding.
This is not a claim of an independent third-party security audit.
